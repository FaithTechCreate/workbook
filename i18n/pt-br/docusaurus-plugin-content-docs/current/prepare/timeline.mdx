# Estabelecendo Seu Cronograma de Ciclo Create.

O planejamento efetivo de um cronograma é crucial para o sucesso de seus projetos
de ciclo 4D. Você pode estar liderando um projeto ou coordenando múltiplos projetos
de sua comunidade. Este guia irá te ajudar a criar cronogramas realistas e flexíveis
que equilibram a urgência com um planejamento cuidadoso.

## Escolhendo uma Duração de Create

Ao planejar a duração do processo de Create, você pode considerar que ele
deve estar relacionado ao tempo estimado que o processo levará para ser concluído.
Se você está liderando múltiplos projetos em sua comunidade, recomendamos fortemente
que você sincronize os Ciclos Create entre os vários projetos e equipes. Isso
facilita a organização de sessões de tutoria e demonstração.

Ao decidir a duração de um projeto, considere os fatores a seguir:

- Complexidade do projeto: projetos mais complexos requerem cronogramas mais extensos.
- Disponibilidade da equipe: considere o tempo de comprometimento que sua equipe pode 
oferecer de maneira realista.
- Urgência da necessidade: algumas comunidades podem requerir uma ação mais rápida.
- Disponibilidade de recursos: certifique-se de que os recursos necessários estejam
disponíveis para um projeto de tal duração.

Opções para duração de projeto:

1. Sprints de curto-prazo (1-4 semanas)
2. Projetos de médio-prazo(1-3 meses)
3. Iniciativas de longo-prazo (3+ meses)

Você pode executar um Ciclo Create de 4 semanas se você está trabalhando em um projeto
a curto-prazo. Alternativamente, você pode executar um Ciclo Create de 12 semanas para
um projeto de médio-prazo. Se você está liderando múltiplos projetos em sua comunidade,
um Ciclo Create de 12 semanas pode ser um bom ritmo que funciona bem para projetos de
curto e médio prazo. Você pode repetir múltiplos Ciclos Create para iniciativas de
longo-prazo.

Exercício: para o projeto de sua escolha, liste as vantagens e as desvantagens para cada
opção de duração. Discuta com sua equipe para chegar a um consenso.

## Desenvolvendo um Roadmap Create

Uma vez que você definiu a duração do seu Create, siga os seguintes passos para
desenvolver um roadmap:

1. Identifique o ritmo para suas reuniões da comunidade e de equipe.
2. Se necessário, mapeie oportunidades adicionais para se reunir.
3. Reserve tempo de preparação para a geração de projetos e formação de equipes.
4. Estabeleça metas para cada estágio do Ciclo 4D.
5. Use ferramentas de gerenciamento de projeto (Trello, Asana, etc.) para visualizar
o roadmap.

## Gerenciando as Expectativas do Cronograma

Para gerenciar efetivamente as expectativas:
- Comunique os cronogramas com clareza para todos os stakeholders
- Forneça atualizações periódicas sobre o progresso e as mudanças
- Seja transparente sobre os desafios e os atrasos
- Celebre o alcançar dos objetivos para manter a motivação

Exercício: crie um plano de comunicação para seu cronograma de projeto, incluindo
a frequência e os métodos de atualização para diferentes grupos de stakeholders.

## Perguntas de Reflexão

- Como nosso cronograma reflete nosso comprometimento aos princípios da tecnologia
redentiva?
- Estamos disponibilizando tempo o suficiente para a condução de Deus em cada estágio
do ciclo 4D?
- Como podemos garantir que nosso cronograma promova um equilíbrio saudável de vida e
trabalho para os nossos membros de equipe?

Lembre-se, apesar de cronogramas serem importantes para o gerenciamento de projetos, eles
devem permanecer flexíveis o bastante para acomodar a liderança do Espírito Santo e 
oportunidades inesperadas para o impacto redentivo.
